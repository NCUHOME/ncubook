# 此间手机端标准页共识稿 v1

## 适用范围

本稿只定义学生端的「此间模块首页」。它不是完整 App 首页，也不是运营后台首页。

产品关系：

```text
宿主校园 App
  -> 底部导航某个 Tab：生存手册 / 此间
    -> 加载此间 Web 模块
```

因此，学生端页面不展示 `/ops`、数据看板、Query Log、知识缺口统计等运营信息。运营台只作为面试演示和内部复盘页保留。

## 页面目标

用户进入「此间」后，需要快速完成三件事：

1. 搜索或询问一个校园问题。
2. 按主题进入指南内容。
3. 发现信息过期时能提交修正。

首页不承担“展示 Agent 能力有多完整”的任务。Agent 能力应该藏在体验背后，通过可追溯来源、弱命中提醒、反馈闭环体现。

## 标准页面结构

### 1. 模块顶部栏

用途：告诉用户这是当前 App 内的一个功能模块，而不是另一个独立 App。

保留：

- 标题：`此间`
- 副标题：`南昌大学生存手册`
- 右侧图标：搜索、更多 / 分享

调整：

- 不再使用“打开导航”的菜单按钮作为主要入口。
- 不展示用户头像式 AI 入口，避免像独立 App 个人中心。
- 如果嵌入宿主 App，返回和宿主导航由 App 处理；Web 模块内部只保留必要操作。

建议文案：

```text
此间
南昌大学生存手册
```

### 2. 搜索 / 问答入口

用途：这是首页第一优先级。用户不是来逛页面的，而是来解决问题的。

保留：

- 搜索输入框
- 发送 / AI 图标按钮
- 快捷问题 chips

调整：

- placeholder 从 `问小家园：转专业流程怎么走？` 改为更通用的 `搜索攻略，或问小家园`
- 快捷问题从运营视角改成学生真实问题。

建议快捷问题：

```text
校园卡丢了怎么办？
绩点怎么算？
宿舍怎么报修？
转专业要注意什么？
```

### 3. 小家园入口卡

用途：把 AI 问答作为此间模块的核心能力，但不把首页做成聊天页。

必须出现：

- 标题：`小家园问答`
- 一句话说明：`问校园流程、生活经验和文档位置`
- 两个信任标签：`来源可追踪`、`重要信息需核实`

不能出现：

- `query log`
- `RAG`
- `检索相似度`
- `知识缺口数量`
- `自动解决率`

这些属于内部产品能力，不属于学生端 UI。

### 4. 分类入口

用途：让用户在不提问的情况下，也能快速进入结构化文档。

首页标准分类固定为四个：

```text
新生
学业
生活
发展
```

每个分类只展示一个短标签，避免首页信息过载：

```text
新生：报到 / 校园卡
学业：绩点 / 选课
生活：食堂 / 报修
发展：保研 / 竞赛
```

当前代码里的 `学业与流程`、`生活与办事` 可以拆成四个入口，这是合理的小改，不是重写。

### 5. 最近更新

用途：给用户一个“内容在维护”的信号。

保留：

- 当前首页的「此刻正在发生」思路
- 3 条更新
- 状态标签

调整：

- 标题改为 `最近更新`
- 删掉过强的运营词，比如 `已回流至知识库`、`需要补白`
- 改成用户能理解的状态：`已更新`、`需核实`、`共建中`

建议三条：

```text
校园卡丢失与补办
绩点与学分计算
宿舍报修流程
```

### 6. 反馈 / 共建入口

用途：让用户知道可以纠错，但不要让他看到后台处理过程。

保留：

- 当前 `校订与补白` 模块

调整文案：

```text
发现信息过期？
提交修正，帮助后续同学少走弯路。
```

按钮：

```text
提交修正
```

## 首页信息优先级

从高到低：

1. 搜索 / 问小家园
2. 小家园问答入口
3. 四个分类入口
4. 最近更新
5. 提交修正

不要把“数据健康度”“知识缺口”“运营指标”放在学生端首页。

## 学生端禁区

以下内容不进入学生端普通页面：

```text
/ops
数据看板
Query Log
Knowledge Gap 数量
同步到飞书状态
自动解决率
平均延迟
检索相似度
内部处理状态
```

它们可以留在：

```text
/ops 内部演示页
飞书多维表格
作品集说明文档
面试 Demo 视频
```

## 视觉规范

沿用当前 `CijianMobile.module.css` 的方向，但做收口：

颜色：

```text
背景：#fbfaf8
卡片：#ffffff
浅底：#f3f2ef
正文：#1a1c1b
弱文字：#63645f
强调绿：#006c49
浅绿：#dff6ec
提醒琥珀：#a15c10
```

组件：

- 圆角最大 8px
- 图标按钮 36px 左右
- 卡片只用于具体内容块，不把整个页面包成大卡
- 中文正文 14-16px
- 卡片标题 15-16px
- 品牌标题可用宋体 / Noto Serif SC
- 不使用斜体
- 不使用渐变球、装饰光斑、营销式大 Hero

## 对当前首页的改动清单

对应文件：

```text
src/pages/index.tsx
src/css/CijianMobile.module.css
```

建议保留：

- `ProductTopbar` 基础结构
- `AskComposer`
- `SectionHeader`
- `contributionBand`
- 当前绿色 / 米白视觉方向

建议修改：

- `healthStats` 从学生端首页移除
- `channels` 从 2 个大入口改成 4 个分类入口
- `liveUpdates` 文案从运营语言改成用户语言
- `bottomNav` 不再作为此间模块自己的完整 App 导航；如果要保留预览环境，应标注为宿主 App 导航，不出现 `运营`
- `ProductTopbar` 去掉菜单按钮语义，改为搜索 / 更多等模块操作
