# 此间设计系统规范

状态：视觉基调已确认，核心页面样张已批准

## 设计意图

此间应当安静、可信、克制，首先服务手机上的提问、查找和长文阅读。它不是文档框架皮肤、后台系统，也不是赛博风 AI 聊天工具。

已批准的视觉方向为“编辑黑白”：纯白画布、黑墨文字、灰色层级、克制边线。编辑气质来自中文排版、留白与内容秩序，不使用暖灰纸色或品牌绿色制造氛围。

## 设计原则

1. **内容先于界面**：长文、地图、表格和作者叙事可以决定版面，不以卡片模板强行覆盖内容。
2. **移动先于桌面**：360、390、430px 是首要审查尺寸；桌面只能扩展空间，不能改变任务路径。
3. **单一主操作**：每个页面只有一个视觉主操作。首页是提问，文章是阅读，搜索页是输入关键词。
4. **渐进显露工具**：页面树、目录、搜索、AI 在需要时唤起，不能常驻抢占正文空间。
5. **可信而不过度装饰**：色彩服务语义与层级；留白、排版和内容本身承担审美。

## 排版

- 界面和正文仅使用 `tokens.json` 的 `fontFamily`。系统黑体承担导航、正文和操作；宋体仅用于首页展示标题、板块标题和文章一级标题，不进入按钮、标签或长篇正文。
- 手机正文默认 `fontSize.body` 和 `lineHeight.body`；不得使用细字重作为常规文本。
- 标题层级通过字号、字重和间距表达，不依赖大面积颜色或装饰线。
- 每页最多使用两类字体角色：编辑型展示标题与无衬线界面/正文。

## 空间与色彩

- 只使用 `tokens.json` 的语义令牌。页面不能直接定义色值、阴影、圆角、字号或间距。
- 页面画布固定纯白。主操作使用黑色，次级层级使用灰色和边线；首版不设置品牌强调色。
- 链接默认使用黑色下划线或明确的行尾箭头，不靠颜色单独表达可点击性。
- 红、黄、绿只表达风险、提示、成功等语义，不作为随意装饰。
- 触控目标默认不小于 44px；图标本身可为 20px 或 24px，但必须置于足够触控区内。

## 明确禁止

- 常驻双侧栏、桌面文档站压缩版、首页瀑布式卡片墙。
- 未经定义的渐变、玻璃拟态、发光效果、虚假的 AI 视觉元素。
- 在同一界面混用 emoji、不同图标库或不同线宽的 SVG。
- 为了整齐而改写文章、拆散表格或去除图文关系。
- 使用米白、泛黄或大面积浅灰作为页面画布；灰色只能用于局部次级表面和分隔。

## 审核标准

每项 UI 变更必须在 360、390、430px 下检查：信息层级、文字可读性、触控面积、长表格、图片/多栏堆叠、抽屉焦点、深色/高对比需求和空状态。

## 已批准的交互底座

- 使用 Radix Primitives 的 Dialog/Sheet、DropdownMenu、Accordion、Tooltip 和 VisuallyHidden，或等价的原生语义实现。
- 引入上述依赖已由本规格批准；新增 UI 库、主题包或 primitive 类型仍需新规格批准。
- 视觉样式必须由此间的令牌和组件实现，不得采用任何全量主题的默认外观。

## 已批准的视觉样张

- 首页：宋体展示标题、居中的提问入口、低权重场景索引。
- 板块页：自由编排的板块导读与线性文档入口，不做卡片墙。
- 长文页：纯白正文、无固定双侧栏，搜索、页面树和 AI 按需出现。
- 三类页面共享同一套黑白灰令牌、细边线、系统图标和间距节奏。
