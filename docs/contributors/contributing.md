# 参与贡献

感谢你愿意为此间补充信息。这里不是要求每个人都写完整长文；很多时候，一条准确的来源、一个过期提醒、一次真实办理经历，就能帮后来的人少绕一圈。

## 最适合提交的内容

| 类型 | 你可以提交什么 |
| --- | --- |
| 信息过期 | 地点、时间、电话、入口、费用、材料发生变化 |
| 信息缺口 | 小家园答不上来，或文档里没有覆盖这个问题 |
| 办事经验 | 真实办理路径、等待时间、容易漏掉的材料 |
| 来源补充 | 官方通知、公告链接、页面截图、窗口说明 |
| 结构优化 | 页面太长、步骤不清楚、标题不好找 |

## 推荐提交格式

请尽量包含这四项：

1. 你要补充或修正的问题是什么。
2. 这条信息来自哪里，例如通知链接、截图、窗口说明或个人办理经历。
3. 它适用于哪个校区、学院、年级或时间段。
4. 你觉得它是已经确认，还是仍然需要再次核实。

## 在线编辑

如果你会使用 GitHub，可以直接在页面底部点击“编辑此页”，在浏览器里修改文档并提交 Pull Request。维护者审核后，内容会自动发布到网站。

## 不会使用 GitHub

你也可以通过更轻量的方式提交：

- 在页面底部点击“没帮助”，填写反馈表。
- 通过小家园追问时，把缺失的信息标记出来。
- 发送邮件到 `book[AT]nchuhome.club`。

## 本地开发

如果你想一次性新增多个页面或调整结构，可以本地运行项目：

```bash
pnpm install
pnpm start
```

文档主要放在 `docs/` 目录下：

```text
docs/
├── onboarding/       # 入学指南
├── academics/        # 学业流程
├── campus-life/      # 校园生活
├── career/           # 升学与求职
└── contributors/     # 共建说明
```

## 写作原则

- 先写结论，再写步骤。
- 遇到流程类内容，尽量拆成“适用对象、准备材料、办理路径、注意事项”。
- 不确定的信息不要写死，可以标注“待核实”。
- 尽量提供来源，避免只有“听说”。
- 不要提交隐私信息、群成员信息或未经授权的个人资料。
