# Acceptance：移动端 AI 知识产品重构

## 视觉审核

- [ ] 首页、搜索、板块首页、长文、AI 抽屉均有 360、390、430px 截图。
- [ ] 每个页面保持单一主操作，文字层级与留白清晰。
- [ ] 没有未批准的颜色、字体、图标、圆角、间距或阴影。
- [ ] 长文的多栏、表格、图片、嵌入和附件在手机可用。
- [ ] 每张基线截图均关联 `approval.md` 中人工批准的路径或链接；agent 不得自行接受截图差异。

## 交互审核

- [ ] 页面树可打开、关闭、返回上级并保持焦点。
- [ ] 关键词搜索只返回命中文档和段落，能跳转锚点。
- [ ] 文档页 AI 默认包含当前页面和章节上下文。
- [ ] 从来源进入文章后能返回原 AI 回答。
- [ ] 使用固定 fixture 和网络 mock 验证：关键词搜索不调用 AI；含多个事实结论的 `grounded` 回答中每个 claim 都有锚点 citation；`insufficient` 回答没有事实性结论。

## 内容与可信度审核

- [ ] 公开页面不依赖读者的 Notion 登录权限。
- [ ] 文档块的语义、内部链接和更新时间被保留。
- [ ] 无来源或低置信 AI 回答不伪装为确定事实。

## 工程审核

- [ ] Typecheck、单测、生产构建通过。
- [ ] 关键页面有视觉回归检查；视觉基线由人工批准。
- [ ] 本次改动可追溯到 requirements、design 与 tasks。
