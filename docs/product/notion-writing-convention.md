# Notion 页面树写作与发布约定

状态：按已批准的 `content-publishing.md` 与 `content-data-contract.md` 执行

## 页面树就是站内层级

- 同步器只读取 `NOTION_ROOT_PAGE_ID` 指定的“南大生存手册”根页面及其后代。
- 根页面的直接子页面发布为站内板块首页；更深层子页面按 Notion 父子关系发布为文档页面树。
- 正文始终写在各自的 Notion 页面中。站内同步完整富块并独立提供访问，读者不会跳转到 Notion。
- 草稿页面放在发布根页面之外；进入发布根页面树即表示参与下一次发布。
- Notion 页面 UUID 是 `Page.id`，父页面 UUID 是 `Page.parentId`。标题、位置或顺序变化不得生成新身份。
- slug 首次发布时生成并持久化；后续改标题或移动页面不得自动改变既有网址。

## 支持的正文结构

| Notion 内容 | 发布块 |
|---|---|
| 普通正文、引用 | `paragraph`、`quote` |
| 一级至三级标题 | `heading` |
| 无序、有序及嵌套列表 | `bulleted-list`、`numbered-list` |
| 提示块 | `callout`，按内容风险映射为 `info`、`warning` 或 `risk`；提示块内的子块递归保留 |
| 分隔线 | `divider`，作为正文中的低干扰水平分隔线呈现 |
| 简单表格 | `table`，保留表头、全部行列和阅读顺序 |
| 图片、文件 | `image`、`file`，必须镜像到站内存储 |
| 分栏 | `columns`，手机按从左到右顺序纵向堆叠 |
| 子页面、页面提及 | `page-link` 或页面树节点，站内解析为内部路由 |
| 校园地图嵌入 | `embed(provider="school-map")` |
| 书签、普通外链 | 保留原 URL 和题注，以正文外链呈现 |

- 标题、段落和可引用表格行使用 `b-<Notion block UUID>` 作为锚点。
- 富文本的粗体、斜体、下划线、删除线、代码、允许颜色和公开链接必须保留。
- Notion 内部页面链接保存目标 `pageId`，不得长期保存 Notion 页面 URL。
- 新块类型只有在数据契约、renderer、手机样例和索引策略同时批准后才能发布。

## 页面元数据

- `school` 第一阶段固定为 `ncu`。
- `sourceUrls`、`campus`、`audiences`、`topics` 与 `riskLevel` 由同步配置按页面 ID 覆盖；未配置时使用安全默认值：空数组与 `riskLevel="normal"`。
- 页面树决定层级，元数据配置不得改变父子关系。
- 敏感或待核实页面必须显式配置风险级别；同步器不得从正文措辞猜测风险。

## 资源与嵌入

- 图片和文件下载后计算校验和，上传到站内控制的存储，再写入 `Asset.publicUrl`。
- 单个资源上限为 50 MiB；ZIP、旧版 Word 与 OOXML 办公附件会保留正文显示名，并使用 ASCII 安全存储键镜像；可执行网页内容仍拒绝发布。
- 大图在阅读器中延迟加载并异步解码，避免阻塞文章首屏；原图仍保留完整标签和细节。
- 不得把 Notion 临时签名 URL 写入发布数据。
- 图片应提供 alt；缺失时记录发布警告，由维护者补齐。
- 第一版只有 `https://school-map.ncuos.com/` 可作为站内嵌入；其余网页只发布标题和外部打开链接。
- 未填写 URL 的空 embed 视为编辑器占位符：发布时记录警告并跳过，不伪造成正文或失效链接。

## 失败、删除与回滚

- 未知块、资源下载失败、内部链接无目标或数据校验失败时，该页面本次发布失败。
- 单页失败继续提供该页最近一次成功版本；首次发布失败的页面不进入导航、搜索或 AI 索引。
- 只有完整版本通过交叉链接、资源、搜索锚点与 schema 校验后，才能切换当前发布指针。
- 页面移出发布根树后只在一次完整成功发布中取消公开；旧 URL 显示内容已更新，不跳到无关页面。
