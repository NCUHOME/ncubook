# 内容发布协议：Notion 编辑，站内独立发布

状态：已确认，2026-07-13

## 目标

Notion 是内容编辑源；公开站是独立、可访问、可检索的发布端。同步目标是保留内容的结构和语义，而不是把页面压缩为 Markdown。

```text
Notion 页面树与富块
  -> 同步器
  -> 站内 page / block-tree / asset / anchor 数据
  -> 富内容 renderer
  -> 公开文章、关键词索引、AI 检索切片
```

## 必须保留的内容语义

- 页面树、标题层级、段落、列表、引用、强调色和链接。
- 表格、图片、图片组、多栏、文件和外部嵌入。
- Notion 内部页面链接，发布后改为站内路由。
- 页面、标题和可引用段落的稳定锚点。
- 来源、更新时间、适用校区和风险级别等可供 AI 使用的元数据。

## 发布端规则

- 图片和附件必须同步到站内可控制的存储；不得将 Notion 临时签名链接直接暴露为长期资源。
- 网页嵌入不是可同步资源。第一版只允许 `school-map.ncuos.com`；其余嵌入默认发布为标题、来源链接和“在新页面打开”的降级块。任何新增 iframe 域名必须先经规格批准，使用 `https`、最小化 sandbox 权限，并通过手机样张与失败状态检查。
- 多栏在宽屏并列，在手机自然纵向堆叠；内容顺序必须可读。
- 表格在手机允许横向滚动，并提供清晰表头；禁止为了适配而丢列或改写数据。
- callout 仅保留其提示语义，采用克制的阅读注释样式；禁止将正文普遍卡片化。
- 不承诺任意 Notion 页面像素级复刻。对原生 API 无法表达的特殊布局，采用少量、明确、可文档化的发布组件约定。
- 未知、不可公开或无法安全发布的块不得静默丢失；同步器必须标记发布失败并保留最近一次成功版本，维护端可见具体块类型和原因。

## 编辑约定

第一版支持普通正文、标题、列表、引用、表格、图文多栏、提示块、图片/文件和网页嵌入。新增发布组件必须同时具备：

1. Notion 中清晰的编辑约定；
2. 同步器的结构化表示；
3. 站内 renderer；
4. 手机与桌面样例；
5. AI 索引策略。

没有以上五项，不能把一次性排版补丁伪装成内容组件。

## 数据与版本契约

公开发布数据的完整结构见 [内容数据契约](content-data-contract.md)。同步、索引、AI 引用和迁移均必须使用该契约；实现阶段不能自行定义字段或稳定 ID 规则。
