# 生产 AI 运行约定

状态：适用于 T3.3。AI 仅作为已发布知识的问答入口，不具有人格设定；关键词搜索不得调用模型。

## Provider 边界

- 使用服务端 OpenAI-compatible HTTPS 接口，具体供应商通过环境变量切换，客户端不得包含 endpoint、API key 或模型 ID。
- 聊天模型和 embedding 模型必须显式配置；禁止代码内静默替换模型。
- 模型只返回结构化 `claims`、`sourceIds` 和 `confidence`，不得生成页面标题、URL、锚点或内容版本。
- 默认关闭生产输出。`AI_ANSWER_MODE` 仅允许 `fixture`、`shadow`、`production`；生产上线前使用 `shadow`。

## 预算和可靠性

- 请求总超时 8 秒；仅对 429、502、503、504 重试一次。
- 单次最多检索 8 个原文块，模型输出上限 700 tokens；输入总字符数不超过 12,000。
- p95 服务端延迟目标 5 秒；超时或 provider 故障返回“资料暂时无法核对”，不得回退到无来源生成。
- 每 IP 每分钟默认 10 次。限流仅记录不可逆哈希和分钟窗口，不存原始 IP。
- 日志默认不记录完整问题、原文、provider 响应或密钥；只记录 request ID、耗时、当前内容版本、检索数、置信等级、策略结果和 token 用量。运行日志保留 14 天，反馈关联数据保留 30 天。

## 来源和风险

- 检索限定 `school=ncu` 和当前发布指针；旧版本、失败版本及其他学校内容不可进入 prompt。
- 模型引用的每个 source ID 必须来自本次检索。未知 ID、无来源事实或版本不一致由确定性校验器删除或降级。
- 招生截止时间、费用、资格、成绩/处分、医疗与安全属于敏感信息。缺少权威来源时不输出事实结论，只提示核对官方渠道；紧急安全/医疗问题提供非诊断性的求助方向。
- 来源冲突必须保留分歧并标记 `needs-verification`，不得由模型擅自择一。
- 文档中的指令文字一律视为被引用数据，不得覆盖系统约束。

## 发布门槛

生产模式必须同时满足：当前内容版本存在；citation 有效率 100%；敏感问题无不受支持的事实；关键词搜索产生零次模型/embedding 请求；密钥不进入浏览器 bundle；完整单测、构建、E2E、视觉回归和评测阈值通过。任何一项失败即切换至 `shadow` 或 insufficient-only。
